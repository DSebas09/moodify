# Moodify — AI-Powered Music Mood Classifier

Moodify is a web app that predicts the **mood of a song** based on its 
audio characteristics using a multiclass classification model trained 
with scikit-learn. The model is based on the **Arousal-Valence model** 
from the psychology of music, the same conceptual framework that Spotify 
uses internally.

Project developed as part of the Introduction to AI course — Master's in Computer Systems.

---

## How does it work?

The user enters the name of a song. The app searches for its audio features
in the processed dataset, runs them through an ML pipeline, and returns one of
four possible moods:

| Mood      | Valence | Energy | Color    |
|-----------|---------|--------|----------|
| Happy     | ≥ 0.5   | ≥ 0.5  | Amarillo |
| Energetic | < 0.5   | ≥ 0.5  | Rojo     |
| Chill     | ≥ 0.5   | < 0.5  | Azul     |
| Sad       | < 0.5   | < 0.5  | Gris     |

## Setup and Installation

### 1. Clone the repository

```bash
git clone https://github.com/DSebas09/moodify.git
cd moodify
```

### 2. Install dependencies with uv

```bash
# Install UV if you don't have it
curl -Ls https://astral.sh/uv/install.sh | sh

# Install project dependencies
uv sync
```

### 3. Prepare the data and train the model
Run the notebooks in order:

```bash
uv run jupyter notebook
```

- notebooks/02_mood_labels_creation.ipynb → genera data/processed/spotify_tracks_mood_clean.csv
- notebooks/03_model_training.ipynb → genera models/mood_classifier.pkl

### 4. Run the app

```bash
uv run python -m app.app
```
    
Open your browser to http://localhost:5000

## Machine Learning model
Three algorithms were evaluated using F1-weighted with 5-fold cross-validation:

| Modelo            | Descripción                                           |
|-------------------|-------------------------------------------------------|
| KNN (k=11)        | Sort by similarity to the k closest songs             |
| Random Forest     | Parallel tree assembly with majority vote             |
| Gradient Boosting | Trees in sequence, each one corrects the previous one |

Features used: `valence`, `energy`, `danceability`, `tempo`,
`acousticness`, `instrumentality`, `loudness`, `popularity`

---

## Dataset
**Source**: [Spotify Tracks Dataset — Kaggle (maharshipandya)](https://www.kaggle.com/datasets/maharshipandya/-spotify-tracks-dataset)

- ~114,000 songs from 125 genres
- Cleaning applied: deduplication by track_id, removal of NaN
- Mood tag derived with rules from the Arousal-Valence model
